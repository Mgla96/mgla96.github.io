Diabetic Retinopothy

Machine Learning

Software: Python

This program predicts whether a patient has a medical condition known as Diabetic Retinopathy based on both categorical(integer) and continuous(float) features. These are the 18 features used below:

0) The binary result of quality assessment. 0 = bad quality 1 = sufficient quality.

1) The binary result of pre-screening, where 1 indicates severe retinal abnormality and 0 its lack.

2-7) The results of MA detection. Each feature value stand for the number of MAs found at the confidence levels alpha = 0.5, . . . , 1, respectively.

8-15) contain the same information as 2-7) for exudates. However, as exudates are represented by a set of points rather than the number of pixels constructing the lesions, these features are normalized by dividing the number of lesions with the diameter of the ROI to compensate different image sizes.

16) The euclidean distance of the center of the macula and the center of the optic disc to provide important information regarding the patient's condition. This feature is also normalized with the diameter of the ROI.

17) The diameter of the optic disc.

18) The binary result of the AM/FM-based classification.

I implemented a decision tree machine learning model for this data with random forests to increase accuracy and threshold splits to prevent overfitting.

Author Predictor

Machine Learning

Software: Python

After training this program with a dataset of articles written by different authors it can predict which of these authors has written an anonymous articles. This program utilizes Multinomial Naive Bayes Multiclass Classification with a bag-of-words representation of the text.

I implemented bigrams and trigrams as well as used the nltk library to incorporate lemmetization as methods to improve my accuracy. The accuracy could be further improved by implementing other features such as article length, using stop words, adding tf-idf weighting, and optimizing current weighting of unigram,bigram,and trigrams. Even more accuracy improvements could potentially be gained by using a word to vector approach which would allow us to implement different machine learning models altogether such as a Support Vector Machines or Neural Networks.

3-Class Linear Classifier

Machine Learning

Software: Python

This program is a basic linear classifier for 3 classes. This model is trained by computing the centroid of each class (e.g. A,B, and C) then constructs a discriminant function between each pair of classes (e.g A/B, B/C, and A/C), halfway between the two centroids and orthogonal to the line connecting the two centroids. For testing it checks the discriminant function between "A or B" and then depending on that answer, decides "A or C" or "B or C".

Ocean Recovery Lab

Software Engineer

Software: Python

Created scripts to assist researchers at the Ocean Recoveries Lab at UC Santa Barbara, which focuses on preserving and recovering ocean ecosystems. These scripts automate the mathematical modeling of their collected data to their specific requirements.

The process of automating workflows and saving people time through software is very enjoyable for me. The researchers use a program called MetaShape in order to convert photos they have taken of coral into 3d models that they can use to measure the growth and health of coral reefs. This process must be done for numerous models which require over 10 steps for each model. Some of these steps can't be done until previous rendering is complete which takes several hours so these researchers will have to be on standby to manually use the computer and continue to the next step after rendering is done. The two programs I created for them automate this process for all models at once. The researchers are only required to do 1 manual step in total for all models to be complete, saving them valuable time.
